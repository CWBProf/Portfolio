Performance‑Tuned SQL

I craft SQL that’s measurably faster—using temp tables and the right indexes where speed gains are on the table. Designed for clarity, reliability, and repeatable wins.

-- Stage raw → temp with narrow types
SELECT ...
INTO #stg
FROM dbo.Raw WITH (NOLOCK);

-- Add a helpful index to accelerate the heavy join
CREATE CLUSTERED INDEX IX_stg ON #stg (CustomerId, OrderDate);

-- Transform into the intake schema
INSERT dbo.Intake (CustomerId, OrderDate, ...)
SELECT s.CustomerId, s.OrderDate, ...
FROM #stg AS s
JOIN dbo.DimCustomer AS d
  ON d.CustomerId = s.CustomerId;
